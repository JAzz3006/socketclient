После создания клиента и сервера создадим докер-файл в корне проекта пишем там что надо.
Затем, в терминале docker build -t socket-server .  где
  socket-server - имя образа
  . - указывает на текущую директорию
Затем, произойдет пострение образа
Затем, запустим контейнер docker run socket-server
Но теперь к серверу (в контейнере) не получится подключиться, так как контейнер изолирован. Надо открыть порт.
Остановим контейнер (Ctrl+C) и снова запустим вот так:
docker run -p 8888:8888 socket-server
        где
       -p привязывает порты между хостом(ваш комп) и контейнером
        слева 8888 - порт локальной машины
        справа 8888 - порт контейнера

После создания докер-образа клиента, запустим оба приложения в контейнерах.
Сначала:
docker run -p 8888:8888 --rm socket-server (получилось!)
где
--rm - это удаление контейнера после запуска, чтобы ненужные контейнеры не пложились на диске
Теперь запустим клиент
docker run --rm socket-client
клиент упал, т.к. клиент не смог достучаться до сервера, потому что хост сервера указан как localhost ,
а в рамках контейнера locathost клиента и localhost сервера разные. Узнаем IP-адрес клиента:
docker container ls - список контейнеров - из него нам надо id контейнера клиента. Затем такое вуду:
docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' <тут id клиента>
теперь это же для сервера:
docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' <тут id сервера>
где
docker inspect - это команда отдает подробную информацию о докер-объекте
После того, как выяснены IP-адреса, надо понять как подружить контейнерного сервера с контейнерным клиентом
Начнем с application.properties в клиенте:
app.server.host=${SOCKET_SERVER_HOST:localhost}
app.server.port=${SOCKET_SERVER_PORT:8888}
это значит, что при наличии переменной среды SOCKET_SERVER_HOST, значение app.server.host
будет подставляться из этой переменной, а если нет, то подставиться localhost. С портом - аналогично
Теперь изменим Dockerfile.
